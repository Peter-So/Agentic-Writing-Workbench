# gpt-image-2 生图一致性规范

说明：电影脚本类项目使用 GPT Image / gpt-image-2 兼容接口。角色四视图统一保存到 `assets/characters/`，分镜和节拍生图如出现角色，必须引用对应角色四视图作为参考图。

## 全局视觉风格锚点

- 画幅比例：16:9，电影短片剧照感。
- 图片尺寸：1K。
- API 分辨率：1536x1024。
- 镜头语言：真实摄影、自然景深、可拍摄布光。
- 风格一致性：同一项目内保持统一色彩基调、镜头质感、颗粒、对比度。
- 禁止：动漫化、海报化、过度磨皮、风格突变、角色五官漂移。

## 生图提示词模板

```text
项目：{项目ID}电影短片
用途：{角色四视图|分镜图|抽帧场景图}
统一风格：{从 style.md 提取}
角色锚点：{从 characters.md 和 assets/characters/character_registry.json 提取，不得改变}
场景锚点：{场景地点、时间、道具、光线}
镜头设计：{景别、机位、镜头运动、构图}
画面内容：{只描述可见画面}
情绪功能：{这一帧承担的叙事情绪}
一致性约束：保持角色身份、服装、道具、色彩锚点一致；禁止风格漂移。
负向约束：不要多余人物、不要文字水印、不要海报排版、不要改变角色五官。
```

## 抽帧三图

每个分镜可生成三张：

1. 第一帧：场景建立，角色位置与情绪起点清楚。
2. 中间帧：动作或冲突最高点。
3. 最后一帧：动作结果、沉默或余味。

## 角色四视图

角色四视图不是 3D 建模文件，而是统一角色参考设定图：

- 正面
- 左侧面
- 背面
- 三分之四侧面

每个主角/重要角色只在 `assets/characters/<角色>/Image/four_view.png` 保存一份统一四视图。后续分镜、节拍、关键帧中出现该角色时，将该四视图加入 `reference_images`。

## 后续 API 接入预留

- model: gpt-image-2
- input: 统一提示词
- reference_images: 角色四视图、风格参考图、场景参考图、重生旧帧图
- output_dir: `projects/writing/novels/<project_id>/assets/images/`
