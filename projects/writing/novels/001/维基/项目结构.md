# 项目结构 Wiki

本文件记录项目结构性目录和文件的用途。系统执行归档、更新和材料路由时优先读取 `project-structure.json`。

| 角色 | 文件 | 用途 | 别名/历史名 |
| --- | --- | --- | --- |
| 基础设定 | `设定/基础设定.md` | 题材、类型基调、核心命题、创作约束等全局基础信息。 | `settings/基础设定.md`, `基础设定.md`, `brief.md`, `logline.md`, `项目简报.md`, `设定.md` |

| 世界观设定 | `设定/世界观设定.md` | 世界规则、时间线、空间规则、制度、禁忌与边界。 | `settings/世界观设定.md`, `世界观设定.md`, `world.md`, `001设定及问题.md`, `穿越设定.md` |

## 结构目录

| 角色 | 目录 | 用途 |
| --- | --- | --- |
| settings | `设定` | 基础设定、世界观、规则等设定层文件。 |
| planning | `规划` | 大纲、情节、节拍、伏笔等规划层文件。 |
| characters | `人物` | 人物档案、人物关系和角色声音材料。 |
| memory | `记忆` | 章节摘要、完成状态和连续性记忆。 |
| style | `风格` | 文风、类型、语言和审美规范。 |
| chapters | `正文` | 确认归档后的章节正文。 |
| outputs | `输出` | 材料包、快照、备份和生成产物。 |
| accepted_planning | `输出/已采纳规划` | 用户确认后的规划类产物快照。 |
| outline_backups | `输出/大纲备份` | 大纲覆盖写回前的自动备份。 |
| invocation_logs | `日志/调用记录` | 创作任务 invocation、节点轨迹、成本和 provider 日志。 |
| skills | `技能` | 项目私有技能。公共技能由类型级技能库提供。 |
| wiki | `维基` | 项目级 Wiki，保存结构索引、稳定规则与项目共识。 |
| references | `资产/参考材料` | 项目专用参考材料。 |

## 路由说明

- 系统写入结构性材料时，应先按 role 查询本文件，再按 aliases/文件名相似度回退。
- 历史项目迁移后，旧文件名会保留在 aliases 中，由本 Wiki 路由到新规范路径。
- 覆盖式修改属于高风险写入，必须经用户显式确认后执行。
